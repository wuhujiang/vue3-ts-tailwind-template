你是资深前端工程师，在这个项目中，当用户**新增或修改路由 / 菜单**时，必须遵循本 Skill。

本项目约定：

- 使用 Vue Router 4
- 路由跳转优先使用 `name`
- 路由配置与菜单、权限关联

---

## 一、路由文件结构与命名

> 具体路径可根据实际项目微调，这里给出通用约定。

- 推荐目录结构：
  - 入口：`src/router/index.ts`
  - 模块：`src/router/modules/*.ts`
- 每个路由模块对应一个业务域，例如：
  - `router/modules/user.ts`
  - `router/modules/system.ts`

路由命名（`name`）规则：

- 使用 **PascalCase**，表达业务语义，例如：
  - 列表页：`UserList`, `RoleList`
  - 详情页：`UserDetail`, `ConfigDetail`

---

## 二、路由配置字段规范

典型单个路由项示例（伪代码）：

- `path`：路径字符串，约定驼峰或中划线小写
- `name`：必填，唯一，使用 PascalCase
- `component`：使用路由懒加载（`() => import('...')`）
- `meta`：包含菜单与权限信息

`meta` 常见字段约定：

- `title`：菜单 / 面包屑显示名称
- `icon`：菜单图标标识（与图标系统对应）
- `keepAlive`：是否缓存页面
- `affix`：是否固定在多标签页中
- `roles`：允许访问该路由的角色列表（仅在有权限系统时使用）

编写路由时，应统一使用上述字段，避免随意新增 meta 字段名称。

---

## 三、跳转规范

### 3.1 使用 `name` 进行跳转（强制）

所有编程式跳转统一使用：

- `router.push({ name: 'RouteName', params, query })`

禁止习惯性硬编码 `path` 字符串跳转，除以下情况：

- 登录页、404、重定向等少数固定基础路由

### 3.2 路由参数

- 动态路由参数通过 `params` 传递，例如：`/user/:id`
- 查询参数通过 `query` 传递，例如：`/user/list?page=1`

获取参数统一使用：

- `const route = useRoute()`
- `route.params.xxx` / `route.query.xxx`

---

## 四、路由与页面 / 视图对应关系

- 一般约定：
  - 一个主要路由对应 `views/` 下的一个页面组件
  - 如：`/user/list` → `views/user/UserList.vue`
- 保持「路由路径 / name / 页面组件命名」语义一致：
  - `name: 'UserList'` 对应组件 `UserList.vue`

当一个路由包含子路由时：

- 父路由组件一般为布局壳或空组件，只负责展示 `<router-view />`
- 子路由实际对应页面组件

---

## 五、路由守卫与权限（如项目中存在）

若项目包含权限控制，与路由的关系约定如下：

- 在全局前置守卫中根据：
  - 登录状态（token/session）
  - `meta.roles` 或类似字段，判断访问权限
- 无权限时跳转到：
  - 登录页 或
  - 403 / 无权限提示页

编写路由时：

- 必须确保 `meta` 中包含权限判断所需信息（如 `roles` 或 `permission` 标识）
- 不在页面组件内部做重复的权限判断逻辑（除非是按钮级别控制）

---

## 六、菜单与路由的联动（如模板中存在菜单系统）

如果左侧菜单基于路由自动生成或部分生成：

- 菜单数据应尽量从路由配置中读取 `meta` 字段生成（例如 `title`、`icon`）
- 新增页面时：
  - 首先新增路由
  - 确保 `meta.title` 等字段正确
  - 如需在菜单中展示，则确保该路由满足菜单生成逻辑条件

禁止在多个位置硬编码同一菜单信息（例如：同时在路由和另一个常量配置文件中重复写同样的标题和路径）。

---

## 七、重定向与通配路由

- 首页重定向：
  - 使用单独路由项配置 `redirect` 到目标页面 `name`
- 404 / 通配路由：
  - 使用 `path: '/:pathMatch(.*)*'` 类型的路由
  - 统一指向 404 页面组件

所有重定向逻辑应尽量使用 `name`，避免未来重构路径时出现大量修改。
