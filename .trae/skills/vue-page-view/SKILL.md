你是资深前端工程师，在这个项目中，当用户在 `views/` 目录下新建或维护 **页面级组件** 时，必须遵循本 Skill。

本项目约定：

- 使用 **Vue 3 + `<script setup lang="ts">`**
- UI：Element Plus + Tailwind CSS
- 样式：`<style scoped lang="scss">`
- 页面类型：典型后台管理页面（列表、详情、编辑、配置等）

---

## 一、页面文件与命名

- 页面文件统一放在：`src/views/` 目录下
- 推荐结构：按业务域分目录，例如：
  - `views/user/UserList.vue`
  - `views/user/UserDetail.vue`
  - `views/role/RoleList.vue`
- 页面组件文件名使用 **PascalCase**，与路由 `name`、菜单文案保持语义一致

页面中可以拆出子组件，子组件放在：

- 同目录的 `components/` 下，例如：`views/user/components/UserSearchForm.vue`

---

## 二、典型「列表页」布局规范

后台项目中，列表页是最常见的页面类型，推荐统一结构如下：

1. **搜索 / 筛选区域**
   - 放在页面顶部，通常使用 `el-form` + Element Plus 表单组件
   - 样式上可使用卡片/白底容器 + Tailwind 间距类
2. **操作栏**
   - 包含「新增」「批量操作」等按钮，放在搜索区域下方或与搜索区域并排
3. **表格区域**
   - 使用 `el-table` 渲染数据列表
   - 列配置、序号、操作列遵循后续表格 skill
4. **分页区域**
   - 放在表格下方，统一使用一个分页组件或 `el-pagination`

页面模板建议结构（伪代码示意）：

- 顶部：标题 / 面包屑（如有）
- 中间：搜索表单 + 操作按钮
- 中下：表格
- 底部：分页

---

## 三、「详情 / 编辑」页面布局规范

详情页或编辑页推荐结构：

1. 页面标题 / 面包屑（可由布局统一处理）
2. 主体表单区域：使用 `el-form` + 合理的栅格布局（配合 Tailwind）
3. 底部操作按钮区域：
   - 例如「保存 / 取消 / 返回」按钮
   - 建议对齐方式统一（例如右对齐）

表单字段较多时：

- 可以通过折叠面板、Tab 或分块标题，划分逻辑模块

---

## 四、页面逻辑拆分与职责

页面组件的职责：

- 负责「拼装」：布局 + 组合子组件 + 调用 composables/store
- 尽量不直接堆叠复杂业务逻辑

推荐模式：

- 列表页：
  - 页面只负责：布局 + 把搜索条件/分页参数绑定到 UI
  - 数据获取、分页、排序、筛选等逻辑统一抽到 `useXxxList()` composable
- 详情/编辑页：
  - 页面只负责：布局 + 表单 UI
  - 数据加载与提交逻辑抽到 `useXxxForm()` 或类似 composable

---

## 五、生命周期与数据加载

统一数据加载时机：

- 列表页：在 `onMounted` 中调用 `fetchList()`，或通过 `useXxxList()` composable 自动执行
- 详情页：在 `onMounted` 中根据路由参数加载详情数据

路由参数获取规范：

- 使用 `useRoute()` 获取 `params` 和 `query`
- 不直接硬编码解析 `location` 等浏览器对象

在页面加载时：

- 必须处理 `loading` 状态（列表 loading / 表单 loading）
- 必须处理错误情况（错误提示、空状态展示）

---

## 六、与路由 / Store / Composable 的关系

- 页面组件与路由：
  - 一般一个主要路由对应一个页面组件
  - 路由 `name` 与页面的语义一致，便于跳转与维护
- 页面组件与 Store：
  - 页面需要跨组件共享状态时可以使用 Pinia store
  - 仅当前页面使用的状态建议保持在页面或 composable 内
- 页面组件与 Composable：
  - 页面级逻辑，如列表管理、表单管理，推荐抽出 `useXxxList` / `useXxxForm` 等 composable

---

## 七、可读性与维护性要求

- 页面文件不宜过长：
  - 模板过大时，优先拆出子组件
  - 逻辑过多时，优先拆出 composable
- 功能模块清晰分段：
  - 搜索相关逻辑一段
  - 列表相关逻辑一段
  - 弹窗/表单相关逻辑一段

只要是后台页面，都应尽量遵循上述结构，以确保模板内所有页面风格统一、易于维护。
